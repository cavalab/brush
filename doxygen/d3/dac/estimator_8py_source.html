20 class BrushEstimator(BaseEstimator):
21  """
22  This is the base class for Brush estimators.
23  This class shouldn't be called directly; instead, call a child class like
24  :py:class:`BrushRegressor <brush.estimator.BrushRegressor>` or :py:class:`BrushClassifier <brush.estimator.BrushClassifier>`.
25  All of the shared parameters are documented here.
26 
27  Parameters
28  ----------
29  mode : str, default 'classification'
30  The mode of the estimator. Used by subclasses
31  pop_size : int, default 100
32  Population size.
33  max_gen : int, default 100
34  Maximum iterations of the algorithm.
35  verbosity : int, default 0
36  Controls level of printouts.
37  max_depth : int, default 0
38  Maximum depth of GP trees in the GP program. Use 0 for no limit.
39  max_size : int, default 0
40  Maximum number of nodes in a tree. Use 0 for no limit.
41  cx_prob : float, default 1/7
42  Probability of applying the crossover variation when generating the offspring,
43  must be between 0 and 1.
44  Given that there are `n` mutations, and either crossover or mutation is
45  used to generate each individual in the offspring (but not both at the
46  same time), we want to have by default an uniform probability between
47  crossover and every possible mutation. By setting `cx_prob=1/(n+1)`, and
48  `1/n` for each mutation, we can achieve an uniform distribution.
49  mutation_options : dict, default {"point":1/6, "insert":1/6, "delete":1/6, "subtree":1/6, "toggle_weight_on":1/6, "toggle_weight_off":1/6}
50  A dictionary with keys naming the types of mutation and floating point
51  values specifying the fraction of total mutations to do with that method.
52  The probability of having a mutation is `(1-cx_prob)` and, in case the mutation
53  is applied, then each mutation option is sampled based on the probabilities
54  defined in `mutation_options`. The set of probabilities should add up to 1.0.
55  functions: dict[str,float] or list[str], default {}
56  A dictionary with keys naming the function set and values giving the probability
57  of sampling them, or a list of functions which will be weighted uniformly.
58  If empty, all available functions are included in the search space.
59  initialization : {"grow", "full"}, default "grow"
60  Strategy to create the initial population. If `full`, then every expression is created
61  with `max_size` nodes. If `grow`, size will be uniformly distributed.
62  validation_size : float, default 0.0
63  Percentage of samples to use as a hold-out partition. These samples are used
64  to calculate statistics during evolution, but not used to train the models.
65  The `best_estimator_` will be selected using this partition. If zero, then
66  the same data used for training is used for validation.
67  batch_size : float, default 1.0
68  Percentage of training data to sample every generation. If `1.0`, then
69  all data is used. Very small values can improve execution time, but
70  also lead to underfit.
71  random_state: int or None, default None
72  If int, then the value is used to seed the c++ random generator; if None,
73  then a seed will be generated using a non-deterministic generator. It is
74  important to notice that, even if the random state is fixed, it is
75  unlikely that running brush using multiple threads will have the same
76  results. This happens because the Operating System's scheduler is
77  responsible to choose which thread will run at any given time, thus
78  reproductibility is not guaranteed.
79 
80  Attributes
81  ----------
82  best_estimator_ : _brush.Program
83  The final model picked from training. Used in subsequent calls to :func:`predict`.
84  archive_ : list[deap_api.DeapIndividual]
85  The final population from training.
86  data_ : _brush.Dataset
87  The complete data in Brush format.
88  train_ : _brush.Dataset
89  Partition of `data_` containing `(1-validation_size)`% of the data, in Brush format.
90  validation_ : _brush.Dataset
91  Partition of `data_` containing `(validation_size)`% of the data, in Brush format.
92  search_space_ : a Brush `SearchSpace` object.
93  Holds the operators and terminals and sampling utilities to update programs.
94  toolbox_ : deap.Toolbox
95  The toolbox used by DEAP for EA algorithm.
96
